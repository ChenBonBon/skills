请把原始损益表科目映射到标准损益表科目。

只输出 JSON object，格式必须是：{"原科目名称1": "标准科目名称1"}。
要求：

1. key 必须来自原始科目列表，不能改写，不能遗漏，不能新增。
2. value 必须来自标准科目列表或 "__IGNORE__"，不能改写，不能新增其它值。
3. 所有原始科目都要映射上。
4. 只做科目名称映射，不要输出金额、行次、计算过程、置信度、解释文本或数组。
5. 一对多时必须选择最接近、最主要的一个标准科目；多对一允许多个 key 映射到同一个 value。
6. 对“加：”“减：”“其中：”“合计”“净额”“总计”“：”等标识要按财务语义判断，不要仅按字面相似度。
7. 原始科目是明细项但标准表没有完全一致项时，映射到最接近的上级或相关标准科目。例如“其中：银行存款”可映射到“货币资金”。
8. 原始科目是大类标题时，优先映射到标准表中同名大类或最接近科目，不要随意映射到金额明细项。
9. 原始科目存在 OCR 噪声、错别字、缺少冒号、全半角差异时，要按财务语义纠正后选择标准科目，但 key 仍必须保持原始写法。
10. "__IGNORE__" 只用于会造成重复计算、且标准科目中没有独立对应项的原始明细项；不要把普通主科目、合计项、净额项、总计项随意映射为 "__IGNORE__"。
11. 请只输出合法JSON object紧凑排版，并做JSON格式校验后返回，不要包含其它多余文字、换行符、缩进空格符。
12. 科目映射应优先保证财务性质一致：收入/收益类不得映射到成本/费用/支出类，成本/费用/支出类不得映射到收入/收益/利润类，不能仅因名称前缀相似而映射。

关于原始科目与标准科目多对一及一对多情况下处理规则：
利润表以原表主科目金额为准，通常不把“其中：”等明细项汇总到上级费用/收入科目：

- 如果原始科目列表中存在对应主科目，则“其中：”明细项映射为 "__IGNORE__"（明细项不一定带“其中：”，也可能是连续出现在某个主科目后面的费用/收入明细）。
- 如果不存在对应主科目，且该明细项是唯一可用金额来源，才允许映射到最接近的标准科目。

当原始利润表项目来自新准则口径而标准科目列表是旧口径时，按以下顺序选择：

1. 先判断该项目本质是营业收入、营业成本、期间费用、投资/公允价值损益、营业外收入、营业外支出、利润总额、所得税还是净利润，不要先按字面包含词匹配。
2. 如果标准科目列表同时有上级科目和更窄的子项，只有原始科目语义明确等于该子项时才选子项；否则选上级科目，避免把宽口径项目误放入窄口径子项。
3. 含“收益”“损益”的项目不必然是收入类；含“损失”“减值”“支出”的项目不必然是经营费用。应结合“加：/减：”、括号说明和项目在利润表中的位置判断其损益方向和营业内/营业外属性。
4. 不要把非日常经营、减值、处置、重组、罚款、捐赠等偶发或非主营项目映射到“其他费用（主营业务相关）”或“其他费用（期间费用）”，除非原始科目明确属于主营业务或期间费用。
