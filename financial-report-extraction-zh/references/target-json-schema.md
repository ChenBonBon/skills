# 目标原始表 JSON

仅在将拼接后的 `vlm_text` 映射为原始表 JSON，或校验该 JSON 形状时，使用本参考文档。

所有输出值都必须是字符串。保留来源中的日期和数字格式；不要规范化日期、删除逗号、转换括号、把 `--` 转为 `0`，也不要补充缺失小数位。

## 资产负债表

```json
{
  "表名": "资产负债表",
  "编制单位": "",
  "日期": "",
  "单位": "",
  "表格数据": [
    {
      "科目名称": "",
      "年初数": "",
      "期末数": ""
    }
  ]
}
```

## 利润表 / 损益表

输出标准表名 `利润表`。

```json
{
  "表名": "利润表",
  "编制单位": "",
  "日期": "",
  "单位": "",
  "表格数据": [
    {
      "科目名称": "",
      "本月数": "",
      "本年累计数": ""
    }
  ]
}
```

## 现金流量表

```json
{
  "表名": "现金流量表",
  "编制单位": "",
  "日期": "",
  "单位": "",
  "表格数据": [
    {
      "科目名称": "",
      "本年金额": ""
    }
  ]
}
```

## 映射规则

- `表名` 必须是受支持的报表类型之一；否则请用户人工确认。
- `编制单位`、`日期` 和 `单位` 如果在 `vlm_text` 中缺失，可以为空字符串。
- 保留 `vlm_text` 中主表行的顺序。
- 如果分组行、标题行、小计行、合计行属于主财务表，保留它们。
- 目标金额字段缺失时使用 `""`。
- 不要输出 `行次`。
- 丢弃目标 schema 之外的辅助列和额外列。
- 排除签章、制表/复核字段、页码、印章、补充说明和 OCR 噪声。

## 人工确认

当无法安全映射时，使用 Markdown 表格，不要输出局部 JSON。

以下情况必须人工确认：

- `vlm_text` 无法拼接成可读内容。
- 无法从明确表名文本识别报表类型。
- 一次上传中出现多张目标报表。
- 无法定位主财务表。
- 无法从 `vlm_text` 映射 `科目名称`。

如果只有金额列无法匹配，仍然输出 JSON，并将无法匹配的金额字段设为 `""`。
