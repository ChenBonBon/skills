# 目标原始表 JSON

仅在将拼接后的 `vlm_text` 映射为原始表 JSON，或校验该 JSON 形状时，使用本参考文档。

所有输出值都必须是字符串。保留来源中的日期和数字格式；不要规范化日期、删除逗号、转换括号、把 `--` 转为 `0`，也不要补充缺失小数位。

## 资产负债表

```json
{
  "表名": "资产负债表",
  "编制单位": "",
  "日期": "",
  "单位": "",
  "表格数据": [
    {
      "科目名称": "",
      "年初数": "",
      "期末数": ""
    }
  ]
}
```

## 利润表 / 损益表

输出标准表名 `利润表`。

```json
{
  "表名": "利润表",
  "编制单位": "",
  "日期": "",
  "单位": "",
  "表格数据": [
    {
      "科目名称": "",
      "本月数": "",
      "本年累计数": ""
    }
  ]
}
```

## 现金流量表

```json
{
  "表名": "现金流量表",
  "编制单位": "",
  "日期": "",
  "单位": "",
  "表格数据": [
    {
      "科目名称": "",
      "本年金额": ""
    }
  ]
}
```

## 映射规则

- `表名` 必须是受支持的报表类型之一；否则请用户人工确认。
- `编制单位`、`日期` 和 `单位` 如果在 `vlm_text` 中缺失，可以为空字符串。
- 保留 `vlm_text` 中主表行的顺序。
- 如果分组行、标题行、小计行、合计行属于主财务表，保留它们。
- 只有当目标金额字段在 `vlm_text` 中确实缺失，或来源单元格为空时，才使用 `""`。
- 如果 `vlm_text` 中某个科目行包含可见数字文本，必须把该值复制到对应目标金额字段。不要在同一行、紧邻换行延续行或 OCR 表格单元格中存在金额值时，将目标金额字段留为 `""`。
- 当来源行分隔符少于预期或发生列偏移时，应结合表头顺序和相邻行对齐金额值，而不是直接丢弃。数字字符串必须按来源原样保留。
- 输出前执行行级完整性检查：主财务表中的每个可见金额 token，都必须出现在同一科目的 JSON 行中，或明确因为属于辅助列/非目标列而被丢弃。
- 不要输出 `行次`。
- 丢弃目标 schema 之外的辅助列和额外列。
- 排除签章、制表/复核字段、页码、印章、补充说明和 OCR 噪声。

## 人工确认

当无法安全映射时，使用 Markdown 表格，不要输出局部 JSON。

以下情况必须人工确认：

- `vlm_text` 无法拼接成可读内容。
- 无法从明确表名文本识别报表类型。
- 一次上传中出现多张目标报表。
- 无法定位主财务表。
- 无法从 `vlm_text` 映射 `科目名称`。
- 某一行有可见金额值，但无法在不猜测的情况下归入目标金额字段。

如果某个目标金额字段在 `vlm_text` 中确实缺失或为空，输出 `""`。如果金额文本可见但无法安全判断目标列，要求人工确认，不要静默输出 `""`。
